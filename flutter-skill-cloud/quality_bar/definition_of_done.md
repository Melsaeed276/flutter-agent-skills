# Definition of Done

## Scope

Applies to features, bug fixes, and refactors.

## Done means

- Correctness: matches requirements and edge cases.
- UI: responsive across target breakpoints; no overflows.
- Accessibility: basic semantics and tap targets are correct.
- Performance: no obvious jank in primary flows.
- Tests: appropriate coverage (unit/widget/integration) and stable.
- Docs: updated links and short rationale where needed.
